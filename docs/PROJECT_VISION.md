# Project Vision

## Why this project exists

This project started from a simple observation.

A runner can collect years of training data but still lack a true understanding of their own evolution.

The goal is to build the coach that a self-coached runner never had.

The system should combine:

- personal history
- scientific training knowledge
- current athlete state
- reasoning capabilities

to provide personalized coaching.

---

# What we are building

Not:

"A chatbot that knows my activities."

But:

"A coach that follows one runner over time."

The difference is fundamental.

The system must progressively understand:

- who the runner is
- how they respond to training
- what makes them improve
- what limits their performance

---

# The Digital Twin Concept

The central object is the Runner Model.

It represents the current state of the athlete.

A real coach does not remember every training session separately.

They maintain an internal understanding of the athlete.

The Runner Model aims to reproduce this.

---

# Role of AI

The LLM is not the coach.

The LLM orchestrates information from:

- athlete data
- analytics
- Runner Model
- scientific knowledge

The system must explain every recommendation.

Trust is essential.

---

# Product Philosophy

The final product should feel like a coach working in the background.

It should proactively detect:

- progress
- fatigue
- problems
- opportunities

The user should not need to ask every question.
